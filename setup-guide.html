Setup guide

Aug 2, 2018

Prerequisites

  • conda installed
  • PostgreSQL server instance running (with the STRING schema dumps loaded)
  • Neo4j server instance running

Setting up the environment

  1. Clone the GitHub repository

     git clone https://github.com/bornabesic/cytokine-graph-db.git

  2. Create a conda environment (from conda_env.yml)

     make env

  3. Activate the conda environment

     conda activate cgdb

Translating STRING PostgreSQL database to Neo4j graph database

  1. Provide credentials for accessing the databases

    • Create credentials.json file in the root directory of the cloned repository. The format should be the following:

        {
      "postgres": {
          "host": "<hostname>",
          "port": "<port>",
          "database": "<database name>",
          "user": "<username>",
          "pw": "<password>"
      },
      "neo4j": {
          "host": "<hostname>",
          "port": "<port>",
          "pw": "<password>"
      }
  }

  2. Run the Python script (translate_db.py)

     python translate_db.py

    The script expects to read the following format from the standard input (stdin):

    • First line: species name (e.g. Homo sapiens)
    • All other lines: protein name (e.g. CCR5)

Example:

input.txt:

Mus musculus
CCR5
CCL5
IL10RA

Executing the script through the terminal:

python translate_db.py < input.txt

Output:

CCR5 <---> CCL5
Writing associations...
1
Writing actions & pathways...
168
CCR5 <---> IL10RA
Writing associations...
1
Writing actions & pathways...

CCL5 <---> IL10RA
Writing associations...
1
Writing actions & pathways...

Done!
