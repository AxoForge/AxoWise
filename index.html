Workflow

1. Collect the data not automated

  • Download the following STRING database dumps:
    • database.schema.v10.5.pdf (121.3 Kb) - Database scheme
    • items_schema.v10.5.sql.gz (4.9 Gb) - part I: the players (proteins, species, COGs,…)
    • network_schema.v10.5.sql.gz (41.3 Gb) - part II: the networks (nodes, edges, scores,…)
    • evidence_schema.v10.5.sql.gz (6.9 Gb) - part III: interaction evidence (but: excluding license-restricted data)
  • Load the dumps into PostgreSQL database

2. Extract the information from the data not automated

  • Derive the information we are interested in (i.e. for Homo sapiens & Mus musculus) by querying the PostgreSQL database:
    • Associations query (.sql)

3. Build a graph database not automated

  • Turn the extracted information into Neo4j graph database (using our defined database scheme)
  • Build a separate graph for each of the species in the same database (add species label to the nodes)
  • Visualize the graph database

4. Use machine learning not automated

  • Do a classfication / functional prediction of cytokines and cytokine receptors
